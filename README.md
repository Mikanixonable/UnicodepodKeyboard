# Unicode 入力アプリ

Unicode 文字をコードチャート状に閲覧し、クリックで文字列を組み立ててコピーできる静的 Web アプリ。
要件の詳細は [unicode-app.md](unicode-app.md) を参照。

## 実行

`index.html` をブラウザで直接開くだけで動きます（サーバー不要）。ダブルクリック、または:

```sh
open index.html   # macOS
```

Chrome・Firefox の `file://` で動作確認済み。GitHub Pages などの静的ホスティングに置いても、もちろんそのまま動きます。

### file:// 対応の仕組み

ブラウザは `file://` から開いたページで ES モジュール（`import`/`export`）や `fetch()` によるローカルファイル読み込みをブロックします。そのため:

- 全 JS は `<script type="module">` ではなく **classic script**（`window.App.*` 名前空間で連携、`defer` で読み込み順を維持）
- Unicode データは JSON を `fetch` せず、`window.UNICODE_BLOCKS = [...]` のように代入する **`.js` ファイルとして `<script>` タグで読み込む**
- 大きい `names.js`（文字名、約800KB）は起動時に読み込まず、詳細モーダルを開いた時などに `<script>` タグを動的に挿入して遅延読み込みする

## 構成

```

js/

  output.js         出力部（挿入/削除/Undo/Redo/コピー/貼り付け/カウント）
  grid.js           入力部の仮想スクロールグリッド（コードポイント・ジャンプ含む）
  blocks.js         ブロック名ヘッダー＋検索プルダウン＋コードポイント入力欄
  modal.js          文字詳細モーダル
  favorites.js      マイリスト群ストア（localStorage）
  history.js        入力履歴ストア（localStorage、新しい順・重複排除）
  menu.js           共通コンテキストメニュー
  app.js            結線・モード切替・マイリスト管理／履歴キーボード
data/
  segments.js       収録範囲（window.UNICODE_SEGMENTS、初期ロード。範囲の単一の真実）
  blocks.js         ブロック定義（window.UNICODE_BLOCKS、初期ロード）
  categories.js     General Category を RLE 圧縮（window.UNICODE_CATEGORIES、初期ロード）
  names.js          文字名（window.UNICODE_NAMES、遅延ロード。CJK/ハングルは JS で算出し除外）
  block_names_ja.js ブロック名の日本語訳（手動管理・生成対象外。詳細はファイル冒頭のコメント参照）
tools/gen_data.py   data/ を再生成するスクリプト（block_names_ja.js は対象外）
```

## 収録範囲

- **面0 BMP**（U+0000–U+FFFF）
- **面1–3**（U+10000–U+323AF）… 楔形文字(Cuneiform)・エジプトヒエログリフなどの歴史的文字、記号、絵文字、CJK 拡張 B–H
- **面14**（U+E0000–U+E01EF）… タグ・字形選択子補助

各面は「最後に割り当てられたコードポイント」まで自動でトリムします（未割り当ての面 4–13、補助私用領域の面 15–16 は除外）。収録範囲は生成時に [data/segments.js](data/segments.js) へ出力され、JS 側（[js/data.js](js/data.js)）がそれを読むため、範囲定義は生成器の一箇所に集約されています。

未割り当て・制御・サロゲートは空セル表示。グリフ非対応文字はフォント依存（豆腐許容。下記フォント切替も参照）。

## フォント切替

入力部タブ右の「フォント：システム / Noto」で、グリフの描画フォントを切り替えられます（設定は localStorage に保存）。

- **システム**: OS 標準フォントで描画（既定）
- **Noto**: `Noto Sans` / `Noto Sans Cuneiform` / `Noto Sans Egyptian Hieroglyphs` / `Noto Color Emoji` など多数の Noto ファミリを列挙し、ブラウザがグリフ単位でフォールバック

Noto は **埋め込まず、ローカルにインストール済みの Noto フォントを参照**します（アプリを軽量・オフライン対応に保つため）。該当する Noto が未インストールの文字は、システムフォントや豆腐にフォールバックします。全 Unicode をカバーする Noto を同梱するとサイズが数十 MB になり静的・軽量の方針に反するため、この方式を採用しています。

## 属性による色分け

文字セルとブロック名は、その文字（またはブロックの主な文字）の Unicode 属性に応じて次の12グループで色分けされます：文字・記号(結合)・数字・句読点・記号・絵文字・区切り・制御・書式・サロゲート・私用領域・未割り当て。ブロックの色は、そのブロック範囲内で最も多くを占める属性（未割り当てを除く）から決定します（`js/data.js` の `dominantGroupForRange`）。凡例はブロック選択パレット上部に表示されます。

## データ再生成

Unicode バージョンを上げる等で再生成する場合（要ネットワーク。Blocks.txt のみ取得）:

```sh
python3 tools/gen_data.py
```

`js/data.js` の `SEGMENTS` と `tools/gen_data.py` の `SEGMENTS` は一致させること。

## マイリストの永続化について

マイリストは `localStorage` に保存されます。初期状態では星アイコンの「お気に入り」が用意されており、必要に応じて追加のマイリストを作成できます。`file://` でも Chrome・Firefox では動作しますが、Safari は `file://` ページでの Web Storage を制限しており保存されない場合があります（[js/favorites.js](js/favorites.js) は `try/catch` で保護しているため、その場合もエラーにはならず「そのセッション中だけ有効」という穏やかな劣化になります）。

## 操作

- 文字をタップ／クリック → 出力部に挿入
- 右クリック／長押し → メニュー（選択中のマイリストへ追加・詳細表示）
- ブロック名 → クリックで**2次元パレット**（複数列のグリッド）が展開し検索・ジャンプ。各項目は属性ごとに色分けされ、上部に凡例を表示
- 入力部の文字セル・ブロック名は**属性（文字/記号/数字/句読点/絵文字/サロゲート/未割り当て等）で色分け**
- ブロック名は**日本語（主）＋英語（小さく併記）**で表示（訳が無い一部の希少な歴史的文字は英語のみ）
- ブロック名の右の入力欄にコードポイント（例 `1F600` / `U+41`）を入力 → その文字へジャンプ＆ハイライト
- タブで「現在」「履歴」「マイリスト」キーボードに切替。マイリストタブでは複数リストを切り替え・作成・削除できる
  - 現在: 出力欄に**今入っている文字**を出現順・重複なしで表示（直接入力・貼り付け・Undo/Redo にも追従）
  - 履歴: 最近入力した文字を新しい順に表示
- 出力部の「貼り付け」ボタン → クリップボードの内容をカーソル位置に挿入
- 入力部タブ右の「フォント：システム / Noto」でグリフの描画フォントを切替

# Unicode Input App — 要件定義書

> このドキュメントは、単独で文脈が読み取れることを目的とした要件のまとめである。
> 次工程（詳細設計）はこの文書のみを参照すれば全体像が把握できるように記述する。
> 未確定事項・設計判断の根拠・却下した代替案も明記する。

---

## 1. 概要 / ゴール

Unicode 文字をブラウザ上で閲覧・検索し、クリックで文字列を組み立ててコピーできる**静的 Web アプリ**。
コードチャート的に文字を一覧しながら、任意の文字を出力欄に入力・編集・コピーする。よく使う文字はお気に入りに登録し、専用キーボードから素早く入力できる。

### 主要ユースケース
- 記号・絵文字・多言語文字を探してテキストに挿入し、コピーして他アプリに貼り付ける。
- 特定文字の詳細情報（コードポイント・名称・ブロック等）を確認する。
- 頻用文字をお気に入り登録し、専用キーボードで反復入力する。

---

## 2. 技術制約

- **静的サイト**。ネイティブの HTML / CSS / JavaScript のみで構成。
- **ビルド不要・バックエンド不要・外部 API 依存なし**（フレームワーク不使用を基本とする。使う場合もビルド無しで動くこと）。
- 配信は静的ファイルのホスティングのみ（GitHub Pages 等）で完結する。
- データ（Unicode 情報）はビルド時ではなく事前生成した静的 JSON として同梱し、実行時に読み込む（§12 参照）。

---

## 3. 全体の見た目・表示要件

- **レスポンシブ**：PC（広い横幅）とモバイル（狭い縦画面）の両方に最適化。
- **ダークモード**：ダークを基調とする。`prefers-color-scheme` を尊重し、可能ならライト/ダーク切り替えトグルも用意（推奨、優先度低）。
- 画面は上下 2 領域構成：上部 = **出力部**（固定表示）、下部 = **入力部**（スクロール領域）。
- 入力部をスクロールしても出力部は常に見えている（出力部は sticky / 固定）。

---

## 4. Unicode 収録範囲【確定・更新】

> 当初は「BMP＋主要記号ブロックのみ」だったが、歴史的文字（例: 楔形文字 U+12000〜）まで
> 網羅したいという要望を受けて拡張した。

- 次の面を収録する（各面は「最後に割り当てられたコードポイント」まで自動でトリム）：
  - **面0 BMP**（U+0000–U+FFFF）
  - **面1–3**（U+10000–U+323AF）: 楔形文字・エジプトヒエログリフ等の歴史的文字、記号、絵文字、CJK 拡張 B–H
  - **面14**（U+E0000–U+E01EF）: タグ・字形選択子補助
- 未割り当ての**面 4–13** と、補助私用領域の**面 15–16** は除外（表示すべきものが無いため）。
- 収録範囲は生成器（`tools/gen_data.py`）が算出して `data/segments.js` に出力し、フロント（`js/data.js`）がそれを読む。**範囲定義は生成器の一箇所に集約**（二重管理なし）。
- **未割り当て・制御文字・サロゲート等「表示できないコードポイント」は空セルとして表示する**（後述 §7）。グリッド上の位置とコードポイントが 1:1 対応する。
- グリフを持たない文字は**フォント依存**（豆腐 □ 許容）。Web フォントの埋め込みは行わない（静的・軽量の要件を優先）。ただし後述のフォント切替で、ローカルにインストール済みの Noto フォント参照へ切り替え可能。

### CJK/ハングルの名称
CJK 統合漢字（拡張含む）・CJK 互換漢字・ハングル音節は名称がアルゴリズムで導出できるため
`names.js` から除外し、ブラウザ側（`algorithmicName()`）で算出する。これにより面2–3の
約9万の漢字名を配信データに含めずに済む。

### 却下した代替案
- 全 17 面を隙間なく連続描画：未割り当ての面 4–13（約70万コードポイント）が巨大な空白域になり無意味なため、面単位でトリムして除外。
- 全 Unicode をカバーする Noto の同梱：数十 MB になり静的・軽量の方針に反するため却下（フォント切替はローカル参照で対応、§下記）。

## 6. 出力部（Output Area）

- 入力された文字を表示する**複数行編集領域**（textarea 相当）。**改行可能**。
- キーボードからの直接入力・編集も可能（グリッドからの挿入と併用）。
- 文字の挿入・削除は**キャレット（カーソル）位置**を基準に行う。
- ボタン群：
  - **コピー**：領域内テキスト全体をクリップボードにコピー。成功時にフィードバック（トースト or ボタン一時変化）を表示。
  - **1 字削除**：キャレット直前の 1 文字を削除（バックスペース相当）。1 コードポイント単位ではなく**1 書記素（見た目の 1 文字）単位**で削除するのが望ましい（絵文字連結列を壊さない。優先度中）。
  - **全削除**：領域を空にする。実行前に確認、または直後に「元に戻す」で復帰可能とする。
  - **元に戻す（Undo）** / **やり直し（Redo）**：操作履歴スタックを内部保持し、入力・削除操作を巻き戻し/やり直しできる。
    - ※原文の「削除や入力をやり直すボタン」を Undo + Redo の 2 ボタンとして解釈。
- **文字数 / コードポイント数**の表示（推奨追加）。
- 空状態ではプレースホルダを表示。

---

## 7. 入力部（Input Area / グリッド）

- **1 行 16 セル**の固定グリッド。コードポイント昇順で左→右、上→下に並ぶ（Unicode コードチャートに準拠）。
- 各セルは**文字（グリフ）を上、コードポイント（U+XXXX）を下**に表示する。
- **表示できないコードポイントは空セル**（枠のみ・淡色）として表示。入力・詳細表示は不可。
- **仮想スクロール（ウィンドウイング）必須**：BMP だけで 65,536 コードポイント = 4,096 行あり、全 DOM 生成は不可。可視範囲＋前後バッファのみ描画する。
- 結合文字は単独表示時に見やすいよう `◌`（U+25CC）等の点線円を添えて表示するのが望ましい（優先度中）。空白系文字は枠やラベルで存在を示す。

### 操作（重要：原文の矛盾をここで解決）
原文では「クリックで入力」と「選択でモーダル表示」が同一操作に割り当てられ衝突する。以下に統一する：

| 操作 | PC | モバイル | 挙動 |
|------|----|---------|------|
| 挿入 | 左クリック | タップ | キャレット位置にその文字を挿入 |
| メニュー | 右クリック | 長押し | コンテキストメニュー（**お気に入り登録/解除**・**詳細を表示**）を開く |
| 詳細への即時導線 | セル下部のコードポイント表示部をクリック | 同左 | 詳細モーダルを開く（補助導線・推奨） |

- 長押し判定はタップとの誤爆を防ぐしきい値（例: 400–500ms、移動許容範囲小）を設ける。
- ホバー時にツールチップ（文字名・コードポイント）を表示（PC、推奨）。
- **キーボード操作対応**（推奨）：矢印キーでセル移動、Enter で挿入、コンテキストキー相当でメニュー。

### 代替案（未採用・要確認なら差し替え）
- 「シングルタップ＝詳細、別ボタンで挿入」案も可能だが、反復入力の快適さを優先し**タップ＝即挿入**を採用。

## 8. ブロック名ヘッダー & ナビゲーション

- 入力部上端に **sticky なヘッダー**を置き、現在スクロール位置にあるコードブロックの**ブロック名（Unicode Block 名）**を表示。
- スクロールに追従して表示ブロック名を更新する。
- ヘッダーを**クリック/タップするとプルダウン展開**：全ブロック（または前後）の一覧を表示。
  - 一覧は**インクリメンタル検索可能**にするのが望ましい（ブロック数が多いため。推奨）。
  - ブロック選択で、そのブロック先頭まで入力部を**スムーズスクロール**して遷移。

---

## 9. 文字詳細モーダル

- コンテキストメニューの「詳細」またはコードポイント表示部から開く。
- 表示内容（上から）：
  1. **文字を大きく表示**。
  2. **コードポイント**：`U+XXXX`（16 進）、10 進、UTF-8 / UTF-16 バイト列、HTML 実体参照（`&#xXXXX;`）。
  3. **文字名**（Unicode 正式名称）、**ブロック名**、**General Category**（分類）等のメタ情報。
- **左右ボタン**で前後の文字のモーダルへ遷移（キーボード ← → にも対応）。遷移対象は**割り当て済み文字**を基本とする（空セルはスキップ）。
- モーダル内から**挿入**・**お気に入り登録/解除**も行える（推奨）。
- Esc で閉じる。フォーカストラップ・フォーカス復帰などアクセシビリティに配慮。

## 10. お気に入り

- コンテキストメニュー（右クリック/長押し）、またはモーダルから**登録/解除**。
- **localStorage に永続化**（後述 §11）。
- 登録済み文字は視覚的に印（★等）を付ける（推奨）。
- お気に入り管理：**解除**・**並べ替え**（ドラッグ、推奨）ができると良い。
- 上限は特に設けないが、多すぎる場合の表示を破綻させない。

## 11. お気に入りキーボード

- 入力部は **[全 Unicode] / [お気に入り] の 2 モードをトグル切り替え**できる。
- お気に入りモードでは、登録済み文字だけを並べた「キーボード」を表示。
- タップ/クリックで、通常グリッドと同様に出力部のキャレット位置へ挿入。
- 空状態（お気に入り未登録）では登録方法を案内するメッセージを表示。
- お気に入りモードでも詳細表示・解除は可能とする。

---

## 12. 永続化 / データ

### 永続化【確定】
- **お気に入りのみ localStorage に永続化**。出力部の入力内容・現在モード・スクロール位置はセッション限り（永続化しない）。
- localStorage スキーマはバージョン付き（将来の移行に備える）。

### Unicode データ
- 必要データ：**ブロック定義（範囲＋名称）**、**各文字の名称・General Category**（詳細モーダル・ツールチップ・検索用）。
- 出典は **Unicode Character Database (UCD)**（`Blocks.txt`, `UnicodeData.txt` 等）から**事前生成した静的 JSON** を同梱する。
- BMP 全体の文字名データは相応に大きいため、**圧縮・コンパクトなエンコード、またはブロック単位の遅延読み込み**を検討（詳細設計で決定）。名称は必要時（モーダル/検索）に解決する方式も可。
- コードポイント → グリフはブラウザのフォントレンダリングに委ねる（データ同梱不要）。

## 13. パフォーマンス

- 入力部は**仮想スクロール**で可視行のみ描画（§7）。
- スクロール中のブロック名更新は `IntersectionObserver` 等で効率化。
- 大きな JSON はメインスレッドを塞がぬよう分割/遅延ロード。
- モバイルでのスクロール追従・長押し判定が滑らかに動くこと。

## 14. レスポンシブ詳細

- **16 列は維持**（Unicode チャート準拠のため列数は可変にしない）。セル幅は `画面幅 / 16` でスケール。
- 狭幅モバイルではセル下のコードポイント表示が入りきらないため：
  - セル下は**下位 2 桁（16 進）等の短縮表示**にし、行左のガター（余白）に共有プレフィックス（例 `U+004_`）を表示するチャート様式に切り替える（推奨）。
  - 完全なコードポイントはツールチップ/モーダルで常に確認可能。
- 広い画面では各セルに `U+XXXX` を完全表示。

---

## 15. アクセシビリティ（推奨・優先度中）

- グリッド/ボタン/モーダルに適切な ARIA ロール・ラベル。
- キーボードのみで主要操作（閲覧・挿入・詳細・お気に入り）が可能。
- モーダルのフォーカストラップと復帰。
- `prefers-reduced-motion` を尊重（スクロールアニメ抑制）。
- コントラスト比の確保（ダーク基調でも WCAG AA 目安）。

---

## 16. エッジケース

- 結合文字・異体字セレクタ・ゼロ幅文字：単独表示時の見え方に配慮（点線円・プレースホルダ）。削除は書記素単位が望ましい。
- RTL 文字・双方向テキスト：出力部での表示崩れに注意。
- サロゲート・非文字（noncharacter, 例 U+FFFE/U+FFFF）：空セル扱い、挿入・詳細不可。
- クリップボード API 非対応/権限拒否時のフォールバック（`execCommand` 等）と失敗フィードバック。

---

## 17. 対象外（Non-goals）

- サーバー・アカウント・クラウド同期。
- 共有 URL・エクスポート機能（将来拡張候補）。
- Web フォント埋め込みによる全文字グリフ保証。
- 全 Unicode 17 面の網羅（§4）。

---

## 18. 詳細設計で確定させる残課題

1. 収録する補助面ブロックの確定リスト。
2. Unicode データ JSON の具体スキーマ・分割方針・圧縮方式・遅延ロード境界。
3. 仮想スクロールの実装方式（自前 or 軽量ライブラリ）と行高・バッファ量。
4. コンテキストメニューの UI 形態（ネイティブ風メニュー / ボトムシート等）。
5. 出力部の Undo/Redo をネイティブ textarea 履歴に乗せるか独自スタックにするか。
6. 検索機能（文字名・コードポイント検索）を入れるか（推奨候補、現状スコープ外）。
7. ライト/ダーク手動トグルの有無。

# WebUIの構造メモ

このアプリは、`index.html` を入口にした classic script 構成で、`window.App.*` 名前空間に機能を分割している。大きくは「データ層」「状態層」「表示層」「接続層」に分けて考えると追いやすい。

### 1. データ層

- [js/data.js](js/data.js): Unicode の基礎データと共通ユーティリティ。コードポイントの範囲判定、分類、グリッド行計算、名称取得、属性色分けの判定を担当する中核。
- [data/segments.js](data/segments.js): 収録範囲の単一の真実。
- [data/blocks.js](data/blocks.js): Unicode ブロック定義。
- [data/categories.js](data/categories.js): General Category のランレングス圧縮データ。
- [data/names.js](data/names.js): 文字名の遅延ロード対象。
- [data/block_names_ja.js](data/block_names_ja.js): ブロック名の日本語訳。

### 2. 状態層

- [js/output.js](js/output.js): 出力欄の状態管理。挿入、削除、Undo/Redo、コピー、貼り付け、文字数カウントを扱う。
- [js/favorites.js](js/favorites.js): お気に入りの永続化と購読。
- [js/history.js](js/history.js): 入力履歴の永続化と購読。

### 3. 表示層

- [js/grid.js](js/grid.js): 「全 Unicode」タブの仮想スクロールグリッド。セル描画、長押し/右クリックメニュー、ブロックヘッダー連動、現在位置へのジャンプを担当する。
- [js/blocks.js](js/blocks.js): ブロックヘッダー。現在表示中のブロック名、ブロック検索、コードポイント入力によるジャンプを担当する。
- [js/modal.js](js/modal.js): 文字詳細モーダル。名称、分類、UTF-8/UTF-16、前後移動、お気に入り操作を担当する。
- [js/menu.js](js/menu.js): 共有コンテキストメニュー。

### 4. 接続層

- [js/app.js](js/app.js): 各モジュールの生成と結線、タブ切り替え、フォント切り替え、出力欄と各ボードの同期を担当する。
- [index.html](index.html): UI の骨格と script 読み込み順の定義。

### 5. 機能名の整理

今後の機能追加では、次の名前で考えると役割がぶれにくい。

- `OutputArea`: 出力編集と履歴管理
- `Grid`: 全 Unicode の可視化と選択
- `BlockHeader`: ブロック選択とジャンプ
- `DetailModal`: 文字の詳細表示と前後移動
- `Favorites`: お気に入りの保存
- `History`: 入力履歴の保存
- `Data`: Unicode データ参照と共通ロジック
- `App`: 画面同士の配線

### 6. 命名ルールの方針

- 表示部は `render*`、状態更新は `set*` / `update*` / `toggle*` / `move*` に寄せる
- 入力欄のコードポイントは `cp`、コードポイント配列は `list`、ブロックは `block`、グループは `group` を使う
- 「現在表示中」を表すものは `current`、「上端に来ている位置」は `top`、「詳細表示」は `modal` に寄せる
- DOM 生成と副作用は各コンポーネント内に閉じ、`app.js` は結線だけにする

この整理を基準にすると、今後の追加は「どの層の責務か」を先に決めてから実装しやすい。
