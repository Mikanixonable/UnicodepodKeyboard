# Unicode 入力アプリ 仕様書

> 現在の UI、機能、名称、データ構成をひとまとめにした正本。
> 実装は [index.html](index.html) と [js/](js/) 配下、データは [data/](data/) 配下を参照する。

## 1. 用語

- 出力部: 文字列を編集・コピーする上段エリア。実装は `OutputArea`。
- 入力部: Unicode を閲覧・入力する下段エリア。実装は `Grid`、`BlockHeader`、`DetailModal`、`MyLists`、`History` で構成する。
- 符号表: 入力部の「全 Unicode」タブ。
- 入力内容: 入力部の「現在」タブ。出力部に入っている文字の一覧。
- 入力履歴: 入力部の「履歴」タブ。最近入力した文字の一覧。
- マイリスト: 入力部の「マイリスト」タブ。複数作成できる保存済み文字リストの総称。
- お気に入り: 初期状態で用意される星アイコン付きの既定マイリスト。
- 文字詳細: `DetailModal` で開く文字情報ダイアログ。
- ブロック名ヘッダー: 現在位置の Unicode Block 名を示す sticky ヘッダー。実装は `BlockHeader`。

## 2. 画面構成

- 画面は上から下へ「出力部」「ブロック名ヘッダー」「入力部」で構成する。
- 全体は中央寄せで、幅は上限値 `--app-width` に収める。
- 出力部と入力部は同じ横幅を持つ。
- 入力部のグリッドは常に 16 列で、各セルは正方形になるよう計算する。
- ブロック名選択のポップアップは画面内に収まるように表示する。

## 3. 出力部

- 多行のテキストエリアで、直接入力・編集できる。
- ボタンは `コピー`、`貼り付け`、`1字削除`、`全消去`、`Undo`、`Redo`。
- キャレット位置を基準に挿入・削除する。
- 書記素単位の削除を優先する。
- 文字数とコードポイント数を表示する。

## 4. 入力部

### 4.1 タブ

- `符号表`: Unicode 全体を一覧するグリッド。
- `入力内容`: 出力部の現在の文字を重複なしで表示する。
- `履歴`: 最近入力した文字を新しい順で表示する。
- `マイリスト`: 保存済みの複数リストを管理する。

### 4.2 符号表

- 16 列の仮想スクロールグリッド。
- 行は Unicode のコードポイント順に並ぶ。
- 各セルは上にグリフ、下にコードポイントを表示する。
- 表示不能なコードポイントは空セルとして扱う。
- 結合文字は点線円付きで見やすく表示する。
- 右クリック/長押しでコンテキストメニューを開く。
- セル下部のコードポイント表示は文字詳細への即時導線になる。

### 4.3 入力内容・履歴・マイリスト

- いずれもクリック/タップで出力部に挿入できる。
- 右クリック/長押しでメニューを開ける。
- マイリストは複数保持できる。
- 初期状態では星アイコンの「お気に入り」が 1 つある。
- ユーザーはマイリストを新規作成・削除できる。
- マイリスト間で文字の登録状況は独立している。

## 5. ブロック名ヘッダー

- 現在のスクロール位置に対応する Unicode Block 名を表示する。
- クリックするとブロック一覧ポップアップを開く。
- 一覧は検索できる。
- ブロック選択でそのブロック先頭へジャンプする。
- ブロック色は Unicode 属性グループに基づく。

## 6. 文字詳細

- 表示内容は文字、コードポイント、10進値、ブロック名、分類、UTF-8、UTF-16、HTML 実体。
- 前後の割り当て済み文字へ移動できる。
- 入力部への挿入ができる。
- 選択中のマイリストを含め、各リストへの追加/解除ができる。

## 7. コンテキストメニュー

- 文字セル、入力内容、履歴、マイリストの文字から開ける。
- 詳細表示、全Unicodeで表示、マイリストへの追加/解除、履歴削除などを提供する。
- 符号表のメニューでは複数マイリストを選べる。

## 8. データ

- `segments.js`: 収録範囲。
- `blocks.js`: Unicode Block 定義。
- `categories.js`: General Category。
- `names.js`: 文字名。大きいため遅延読込する。
- `block_names_ja.js`: ブロック名の日本語訳。
- `js/data.js`: コードポイント判定、名称解決、色分け、行計算などの共通ロジック。

## 9. 永続化

- 出力部の内容、現在タブ、スクロール位置は永続化しない。
- マイリストのみ `localStorage` に保存する。
- 既定マイリストは保存データに含める。
- 旧 `favorites` / `mylist` 形式は読み込み時に移行する。

## 10. フォント

- 表示フォントは `システム` / `Noto` を切り替えられる。
- `Noto` はローカルにインストール済みのフォントを参照する。
- フォント設定は保存される。

## 11. 実装対応

- `OutputArea`: 出力部。
- `Grid`: 符号表グリッド。
- `BlockHeader`: ブロック名ヘッダー。
- `DetailModal`: 文字詳細。
- `MyLists`: マイリスト群。
- `History`: 入力履歴。
- `Menu`: コンテキストメニュー。
- `app.js`: 結線、モード切替、同期。

## 12. 変更時の方針

- UI 名称を変えるときは、この仕様書の用語を先に更新する。
- 機能を増やすときは、出力部、符号表、入力内容、履歴、マイリスト、詳細、ヘッダーのどこに属するかを先に決める。
- レイアウトを変えるときは、出力部と入力部の横幅を揃える前提を崩さない。
